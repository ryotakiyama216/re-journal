# Git 操作ガイド

今後の開発で利用できるように、今回使用した基本的な Git コマンドをまとめました。

## 基本的なワークフロー

新しい変更を GitHub に反映させる際の標準的な手順です。

### 1. 変更の確認
現在のファイルの変更状態を確認します。
```bash
git status
```

### 2. 変更をステージングに追加
コミットしたいファイルを準備エリア（ステージング）に追加します。
- すべての変更を追加する場合：
  ```bash
  git add .
  ```
- 特定のファイルだけ追加する場合：
  ```bash
  git add <ファイル名>
  ```

### 3. コミット（保存）
ステージングに追加した変更を、メッセージと共にローカルリポジトリに記録します。
```bash
git commit -m "ここに変更内容のメッセージを書く"
```

### 4. GitHub へプッシュ
ローカルのコミットを GitHub（リモートリポジトリ）に送信します。
```bash
git push origin main
```

---

## その他の便利なコマンド

### リモートの変更を取り込む
GitHub 上にある最新のコードをローカルに反映させます。
```bash
git pull origin main
```

### コミット履歴の確認
これまでのコミット履歴を表示します。
```bash
git log --oneline
```

### 変更の取り消し（ステージング前）
ファイルを編集前の状態に戻したい場合。
```bash
git restore <ファイル名>
```
